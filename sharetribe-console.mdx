O **Sharetribe Console** é o painel admin web onde as chefes da V1H gerem o marketplace **sem programador**.

**URL**: `https://console.sharetribe.com/o/v1h4/m/v1h4-test/`

## Estrutura geral

Sidebar dividida em **3 zonas**:

### 1. Home (top)
- V1H — Venue1Hub (nome do marketplace)
- Toggle `test` ↔ `live` (ambientes separados)
- Botões: **View marketplace**, **Copy changes to...**, **Go live**

### 2. Manage (gestão diária)
- Users
- Listings
- Transactions
- Reviews

### 3. Build (configuração do produto)
- General
- Content
- Design
- Users
- Listings
- Transactions
- Monetization
- Integrations
- Advanced

### 4. Suporte (bottom)
- Help Center
- Contact support

## ⚙️ Ambientes (test vs live)

| | test | live |
|---|---|---|
| URL admin | `console.sharetribe.com/o/v1h4/m/v1h4-test/` | `console.sharetribe.com/o/v1h4/m/v1h4-live/` |
| Dados | "Test data" (visíveis com badge laranja) | Dados reais |
| Stripe | Test keys (`pk_test_...`) | Live keys (`pk_live_...`) |
| Sandbox? | ✅ tudo seguro | ⚠️ produção real |

**Botão "Copy changes to..."** — copia configuração (Build) de test → live. **Não copia dados** (users, listings, transactions ficam isolados).

**Botão "Go live"** — promove o ambiente test atual para live (use com cuidado!).

## 📋 Manage — Users

**URL**: `/v1h4-test/users`

Lista todos os utilizadores (atualmente 19 em test). Mostra:
- Avatar
- Nome
- N° de listings
- Data de criação

### Filtros
- Search por nome / email / ID
- Filter por estado, role, etc.

### Ações por utilizador
Click num user → modal/página com:
- **Overview**: Bio, Nome, Email, Stripe account ID, User ID
- **Botão "Email {User}"** — manda email manualmente
- **Listings** count + link
- **Transactions** count + link

### Menu de ações (3 pontos)
- **Verify email** — força verificação manual
- **Log in as user** — entra como o user (suporte). Mostra `LimitedAccessBanner` no v1h durante a sessão
- **Ban user** — bloqueia conta
- **Delete user** — elimina permanentemente

### Download .csv
Exporta lista completa de users para Excel.

## 📋 Manage — Listings

**URL**: `/v1h4-test/listings`

Lista todos os anúncios (atualmente 40 em test). Mostra:
- Thumb
- Título
- Preço
- Data criação
- Badge de estado: Published / **Draft** (cinza) / Pending Approval / Closed

### Click num listing
Abre detalhe:
- **Summary**: título, descrição, preço, localização
- **Custom fields**
- **Photos**
- **Author**

### Menu de ações
- **Close listing** — esconde do marketplace (não apaga)
- **View listing as the author** — vê como o anfitrião vê (preview)
- **Delete listing** — elimina permanentemente

### Aprovar listings pending
Se tens "Listing approval" ativo (em Build → Listings → Settings), novos listings ficam `Pending` até admin aprovar aqui.

## 📋 Manage — Transactions

**URL**: `/v1h4-test/transactions`

Lista todas as transações (atualmente 66 em test). Mostra:
- Thumb do listing
- Título do listing
- Customer / Provider
- Preço
- **Last transition** (Confirm payment / Inquire / Complete / Accept / etc.)
- Created / Last activity

### Click numa transação
Abre detalhe completo:
- **Summary**: booking status (Pending/Accepted/Completed), booking period
- **Participants**: avatares de Customer e Provider
- **Timeline**: histórico de transições
- **Next possible transitions** (com ações):
  - Operator accept
  - Operator decline
  - Outros conforme processo + estado atual

### Menu de ações
- **Add a review** — admin pode adicionar review manualmente
- (Outros conforme processo)

### Operator transitions
O admin (operator) pode forçar transições. Útil para:
- Aceitar manualmente uma reserva travada
- Cancelar uma transação problemática
- Resolver disputas

## 📋 Manage — Reviews

**URL**: `/v1h4-test/reviews`

Lista todas as reviews (atualmente 23 em test). Mostra:
- Avatares (autor + subject)
- "X reviewed Y"
- Rating em estrelas
- Badge `Pending` (se ainda não publicada)

### Click numa review
- **Summary**: "X left a review for Y"
- Data + hora
- Rating
- **Texto da review**
- Review ID

### Menu de ações
- **Delete review** — elimina permanentemente

⚠️ Reviews não podem ser editadas, apenas eliminadas.

- Cores

## 🚦 Workflow típico

### Adicionar nova categoria
1. Build → Listings → Categories → Add category
2. Define key + label PT/EN
3. (Opcional) Build → Listings → Custom fields → adiciona campos só para essa categoria
4. Atualiza `src/config/configListing.js` no código frontend
5. `flex-cli search set` para indexar
6. Commit + push

### Aprovar listing pending
1. Manage → Listings → Filter: Pending Approval
2. Click no listing → revê
3. Approve (no menu de ações)

### Resolver disputa
1. Manage → Transactions → filter: Disputed
2. Click → analisa as mensagens
3. Operator transition: Complete (a favor do customer) ou Cancel (a favor do provider)

### Banir user fraudulento
1. Manage → Users → encontra user
2. Menu: Ban user → confirma
3. (Opcional) Delete user

### Mudar template de email
1. Build → Transactions → Email notifications
2. Seleciona o email (ex: "Booking accepted")
3. Edita HTML / texto
4. Save → próximos emails usam o novo template

## 🔑 Quem tem acesso ao Console

- Apenas **administradores Sharetribe** convidados (não é público)
- Cada admin tem login email/password + 2FA opcional
- **Pendente para a V1H**: convidar emails oficiais da empresa (admin@venue1hub.com em vez de pessoais)

## 🆚 Console vs Código

| Faz-se no **Console** | Faz-se no **Código (VS Code)** |
|---|---|
| Aprovar listings | Adicionar nova página |
| Banir users | Mudar layout |
| Resolver transações | Adicionar componente |
| Editar templates de email | Adicionar custom field (parte 2 — em config) |
| Mudar logo / cores | Mudar lógica de notificações |
| Mudar comissão | Adicionar endpoint API |
| Aceitar refunds | Customizar UX |
| Editar Pages CMS | Mudar SDK calls |

Resumo: **Console = gestão e configuração** / **Código = funcionalidades e UI**.

## Botões importantes no topo do Console

- **View marketplace** — abre o site público numa aba nova
- **Copy changes to live** — copia config test → live (não dados)
- **Go live** — promove env (irreversível, cuidado!)
