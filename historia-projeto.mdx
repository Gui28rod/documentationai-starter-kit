# História do Projeto

Cronologia do desenvolvimento da V1H baseada no histórico Git, desde o fork do template Sharetribe até hoje.

## Ponto de partida

O projeto V1H **não foi construído do zero**. Partiu do [Sharetribe Web Template](https://github.com/sharetribe/web-template) (open-source, MIT license) versão 10.14.0+. Foi feito o fork e depois adaptado em VS Code.

O que veio do **Sharetribe** (não foi programado pela V1H):
- Sistema de transações, pagamentos via Stripe Connect
- Mensagens entre cliente e anfitrião
- Edit Listing Wizard base (Details, Pricing, Availability, Photos)
- ProfilePage e ProfileSettingsPage base
- AuthenticationPage com social logins
- PageBuilder e SectionBuilder (motor universal de páginas)
- Hooks Redux (auth, marketplaceData, user, etc.)
- Sharetribe SDK + Integration SDK

O que foi feito **localmente em VS Code** sobre o template:
- Adaptação visual (cores, fontes, identidade V1H)
- 9 categorias V1H + ícones próprios
- Sistema de favoritos completo
- Sistema de seguidores
- Notificações (5 tipos de toasts + crons de email)
- Tradução automática de conteúdo do utilizador
- Modo escuro
- Multi-booking + long-term booking customizados
- Páginas customizadas (Como Funciona, Destacar Anúncio, Sobre Nós…)
- Mobile category ticker, mobile filters
- PWA + service worker
- Cookie consent GDPR
- Inúmeros endpoints customizados (`server/api/*`)
- 3 cron jobs (`server/jobs/*`)

## Workflow de desenvolvimento

O projeto é desenvolvido em **VS Code** localmente:

1. Programador edita `.js`, `.css`, `.json` na máquina
2. Roda dev server (`yarn run dev`)
3. Testa em `localhost:3000` contra o ambiente Sharetribe `v1h4-test`
4. `git commit` + `git push` para `Gui28rod/v1h---testes-dev` (GitHub)
5. Mudanças vão para o branch `main`

A versão **publicada** em produção será um deploy do branch `main` (provavelmente para Heroku ou AWS), apontando para o ambiente Sharetribe **Live** com Stripe **Live**.
