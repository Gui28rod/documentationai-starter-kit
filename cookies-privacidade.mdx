A V1H tem banner e modal GDPR-ready com 4 categorias separadas. O sistema permite ao utilizador aceitar ou recusar cada categoria individualmente, e o consentimento é persistido em localStorage para não ser pedido em cada visita.

## Ficheiro principal

`src/components/CookieConsent/CookieConsent.js` (~320 linhas).

## 4 categorias

Cada categoria agrupa cookies com finalidade semelhante. O utilizador pode aceitar ou recusar as categorias opcionais; a categoria essencial está sempre ativa porque é necessária para o funcionamento do site.

| Categoria | Descrição | Pode recusar? |
|---|---|---|
| **Essential** | Sessão, login, segurança | Sempre ativo (necessário) |
| **Preferences** | Locale (PT/EN), modo escuro, recently viewed | Sim |
| **Analytics** | Google Analytics 4 (tráfego, comportamento) | Sim |
| **Marketing** | Pixels Meta / Google Ads (remarketing) | Sim |

## Banner inicial

O banner aparece na primeira visita ao site, ou quando é lançada uma nova versão do sistema de consentimento que obriga a nova aceitação. Está posicionado em slide-in no fundo da página.

Aparece na **primeira visita** ou quando o utilizador faz reset de versão.

Conteúdo:
- Texto resumido em PT ou EN (`TEXT.pt` / `TEXT.en`)
- Botões: **"Aceitar tudo"**, **"Recusar tudo"**, **"Personalizar"**

## Modal de preferências

O modal de preferências é a interface detalhada onde o utilizador pode escolher individualmente que categorias aceita. Pode ser acedido a partir do banner ou do link "Definições de cookies" no footer.

Aparece quando o utilizador carrega "Personalizar" no banner, ou pelo footer ("Definições de cookies").

Conteúdo:
- Lista das 4 categorias
- Toggle por categoria (custom `<Toggle>` component com `aria-checked` acessível)
- Texto explicativo de cada uma
- Botão "Guardar preferências"

## Storage

A escolha de consentimento é guardada em localStorage, não num cookie real. O campo inclui a versão do sistema de consentimento para permitir forçar nova aceitação quando a política mudar.

- `localStorage.v1h_cookie_consent` — JSON com versão + timestamp + escolhas

## Versioning

```js
const CONSENT_VERSION = 1;
```

Quando se adiciona um novo tracker ou se muda a tabela de cookies, incrementa-se este número. Resultado: todos os utilizadores são forçados a re-consentir.

## Gating do Google Analytics

O Google Analytics só é carregado após o utilizador aceitar a categoria "analytics". Se recusar, o script não é injetado e nenhum dado de tracking é enviado.

Ao **aceitar analytics**:
- Se `!window.gtag` (GA não injetado) — recarrega a página para injetar
- Se já injetado — ativa eventos

Ao **recusar analytics**:
- GA script não é injetado de todo
- Eventos não são enviados

## Hook global

```js
window.__v1hOpenCookiePrefs()
```

Permite a qualquer componente (especialmente o footer) abrir o modal de cookies sem precisar de instanciar o componente diretamente.

## Event dispatch

Quando o utilizador muda consentimento:
```js
window.dispatchEvent(new CustomEvent('v1h:consentChanged'))
```

Outros scripts podem escutar este event e reagir (ex: ativar/desativar pixels Meta).

## Pendências GDPR (antes do go-live)

Ver [Pendências para Go-Live](pendencias-para-go-live):

- Atualizar Termos de Serviço com:
  - NIF da empresa
  - Capital social
  - Número de registo na conservatória
  - Percentagem de comissão V1H
  - DPO nomeado e morada/contacto
- Atualizar Política de Privacidade com mesma info
- Confirmar morada oficial da empresa
- Atualizar **tabela de cookies** se forem adicionados novos trackers (Meta Pixel, etc.)

**Atenção**: DL 7/2004 implica multa se Termos/Privacidade estiverem desatualizados quando o site for público. **Bloqueante** de go-live.

## Estrutura visual

O banner tem slide-in animado a partir do fundo, com fundo escuro e texto branco. O modal está centrado na página e adapta-se ao modo claro/escuro.

Banner: slide-in bottom, fundo escuro com texto branco, botões à direita.
Modal: centrado, fundo branco/escuro consoante modo, lista vertical de categorias.
