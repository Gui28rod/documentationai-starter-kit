Esta página descreve a **experiência do utilizador** ao gerir reservas. Para os processos Sharetribe + estados + transições em detalhe técnico, ver [Processos de Transação](processos-transacao).

## InboxPage

**URL**: `/inbox`
**Ficheiro**: `src/containers/InboxPage/`

### Tabs

| Tab | O que mostra |
|---|---|
| **Sales** | Transações em que o utilizador é **anfitrião** (recebeu reservas) |
| **Orders** | Transações em que o utilizador é **cliente** (fez reservas) |

### Funcionalidades

- **Lista** ordenada por data (mais recente primeiro)
- **Filtros**:
  - `InboxSearchForm` — busca por texto
  - `InboxSortBy` — ordenar por data, estado, valor
- **Badge de estado** em cada transação (Pendente, Aceite, Concluída, etc.)
- **Mensagens integradas** — chat entre cliente e anfitrião dentro de cada transação
- **Botão de ação contextual** ao clicar numa transação (Aceitar / Recusar / Pagar / Cancelar conforme estado)

### Badge no Topbar

O número vermelho no ícone Inbox do topbar conta:
- Sales novos (anfitrião não viu ainda)
- Orders novos
- Direct messages não lidas

Cap visual "4+" se > 4. Ver [Topbar](topbar) + [Notificações](notificacoes-completas).

## Botões e funcionalidades adicionais

### Lista de espera (Waitlist)

Quando um listing está esgotado, aparece botão **"Juntar à lista de espera"** no OrderPanel:

- Componente: `src/components/OrderPanel/WaitlistButton/`
- Endpoint: `POST /api/waitlist` com `{ email, listingId, date? }`
- Envia **confirmação** ao utilizador
- Envia **aviso ao admin** (email)
- Quando o anfitrião disponibilizar slots, mail automático aos waitlisted

### Multi-booking auto-accept

Para o processo customizado **`default-multi-booking-v10`**:

- Cliente seleciona múltiplas datas/horas no OrderPanel
- Cria **batch de transactions simultaneamente** (uma por slot)
- Anfitrião configurado com **auto-aceitação** — não precisa aprovar cada uma manualmente
- Cliente recebe confirmação imediata após pagamento

Ver [Processos de Transação](processos-transacao) para detalhe técnico.

### Long-term booking

Para reservas de meses/anos (processo `default-long-term-booking-v10`):

- Estado adicional `LONG_TERM_PENDING_APPROVAL` — anfitrião tem 90 dias para confirmar
- Timer no inbox indica quanto falta
- Estados finais especiais (RESERVED, DECLINED, EXPIRED, CANCELLED)

### "Adicionar ao calendário"

Após uma reserva ficar `ACCEPTED`, aparece botão na TransactionPage:

- **Google Calendar** ✅ ativo — abre nova aba com evento pré-preenchido
- **Apple Calendar + Outlook** ocultos via flag `SHOW_APPLE_OUTLOOK=false` (memória [calendar button done](decisoes-importantes))
- Componente: `src/components/AddToCalendarButton/`

### Partilha de orçamento

Botão **"Partilhar orçamento"** no OrderPanel envia link público com:

- Preço calculado
- Datas
- Detalhes do espaço
- Útil para o cliente partilhar com colegas/chefes antes de fechar

Link válido por 30 dias (configurável).

### Dashboard do anfitrião

Componente: `HostDashboard` (visível no `ManageListingsPage`).

Mostra ao anfitrião:
- Total de reservas no mês
- Revenue acumulado
- Pending payouts
- Rating médio dos seus listings
- Reviews recentes

Dados via `GET /api/host-stats`. Ver [Endpoints](endpoints).

## Padrão de ações por estado

| Estado | Cliente vê | Anfitrião vê |
|---|---|---|
| `PENDING_PAYMENT` | **Pagar agora** | À espera de pagamento |
| `PREAUTHORIZED` | À espera de aceitação | **Aceitar** / **Recusar** |
| `ACCEPTED` | "Adicionar ao calendário" | "Confirmado" |
| `COMPLETED` | **Avaliar** | **Avaliar** |
| `CANCELED` | "Cancelado" | "Cancelado" |

Para a lista completa de estados + transições, ver [Processos de Transação](processos-transacao).
