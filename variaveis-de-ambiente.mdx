Configuração da aplicação via `.env` (em dev) ou variáveis no servidor (Heroku/AWS em produção).

## Ficheiros .env

| Ficheiro | Quando se aplica |
|---|---|
| `.env` | Base, sempre carregado |
| `.env.development` | Quando `REACT_APP_ENV=development` |
| `.env.test` | Quando `REACT_APP_ENV=test` (npm test) |
| `.env-template` | Template para copy → não usado em runtime |

## 🔑 Sharetribe Marketplace SDK

| Variável | Exemplo | Onde se usa |
|---|---|---|
| `REACT_APP_SHARETRIBE_SDK_CLIENT_ID` | `7196197a-...` | Frontend SDK + SSR |
| `SHARETRIBE_SDK_CLIENT_SECRET` | `75279c4a59ca...` | Server-side privileged calls |
| `REACT_APP_SHARETRIBE_SDK_BASE_URL` | `https://flex-api.sharetribe.com` | API base URL |

**Obrigatórias**. Sem estas o site não arranca.

## 🔧 Sharetribe Integration SDK (operator)

| Variável | Exemplo |
|---|---|
| `SHARETRIBE_INTEGRATION_CLIENT_ID` | `2d3712aa-...` |
| `SHARETRIBE_INTEGRATION_CLIENT_SECRET` | `57b822a30973...` |

**Usadas por**: cron jobs (`notify*Job.js`), endpoints que precisam de acesso operator (recompute-listing-rating, dismiss-alerts).

⚠️ **Pendente troca para Live em produção** (atualmente são creds do test env).

## 🚀 Server / Deploy

| Variável | Exemplo |
|---|---|
| `PORT` | `3000` |
| `REACT_APP_ENV` | `production` / `development` / `test` |
| `BASIC_AUTH_USERNAME` | `sharetribe` (opcional, staging) |
| `BASIC_AUTH_PASSWORD` | `secret` (opcional, staging) |
| `REACT_APP_SHARETRIBE_USING_SSL` | `true` em produção |
| `SERVER_SHARETRIBE_TRUST_PROXY` | `true` em Heroku |
| `REACT_APP_SHARETRIBE_REDIRECT_SSL` | `true` em produção |
| `REACT_APP_CSP` | `report` ou `block` |
| `REACT_APP_SENTRY_DSN` | (opcional para error tracking) |

## ⏰ Cron Jobs (override)

| Variável | Default | Função |
|---|---|---|
| `FAVORITE_DIGEST_CRON` | `0 8 * * *` | Schedule do digest de favoritos |
| `SAVED_SEARCH_NOTIFY_CRON` | `*/5 * * * *` | Schedule notificador pesquisas |
| `EXTRA_ALERTS_CRON` | `*/5 * * * *` | Schedule extra alerts |
| `DISABLE_FAVORITE_DIGEST` | (não definida) | `true` para desativar |
| `DISABLE_SAVED_SEARCH_NOTIFIER` | (não definida) | `true` para desativar |
| `DISABLE_EXTRA_ALERTS` | (não definida) | `true` para desativar |

## 🛠️ Cache e Performance

| Variável | Exemplo |
|---|---|
| `TTL` | `60` (cache geral) |
| `TTL_ASSETS_BY_VERSION` | `86400` (1 dia) |
| `VERBOSE` | `true` para logging detalhado |
| `PREVENT_DATA_LOADING_IN_SSR` | `true` para desativar SSR data loading |

## ⚠️ Pendências de produção

Ver [Pendências para Go-Live](pendencias-para-go-live).

| O quê | Estado atual | Para go-live |
|---|---|---|
| Stripe keys | `pk_test_*` | `pk_live_*` |
| Integration SDK | test env | Live env |
| GA4 | configurado em test | "Copy to live" no Console |
| MARKETPLACE_ROOT_URL | `localhost:3000` | domínio real |
| Stripe Secret Key | no Sharetribe Console (test) | Console (live) |

## Mandatory check no boot

`server/index.js` verifica no arranque:

```js
const MANDATORY_ENV_VARIABLES = [
  'REACT_APP_SHARETRIBE_SDK_CLIENT_ID',
  'SHARETRIBE_SDK_CLIENT_SECRET',
  'REACT_APP_MARKETPLACE_NAME',
  'REACT_APP_MARKETPLACE_ROOT_URL',
];
```

Se alguma faltar → `process.exit(9)`. As outras são opcionais (com fallback ou feature flag).
