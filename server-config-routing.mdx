Detalhe técnico das fundações do projeto.

## Server (`server/`)

### Entry point

| Ficheiro | Função |
|---|---|
| `index.js` | Entry principal. Configura Express com middlewares (Helmet, compression, SSL enforce), carrega env, inicia Passport (auth), SSR renderer, mapeia rotas. Arranca cron jobs no final. |
| `apiRouter.js` | Express router com cerca de 23 endpoints prefixados `/api/*`. Tudo em [Endpoints](endpoints). |
| `auth.js` | Middleware basic auth (username/password) para staging |
| `csp.js` | Content Security Policy com **nonce por request**; whitelist Sharetribe, Mapbox, Stripe e Google CDN |
| `dataLoader.js` | Carrega dados server-side para SSR (Redux store, translations, hosted config) |
| `renderer.js` | Renderiza app com Helmet (SSR); injeta assets dinâmicos no HTML template |
| `log.js` | Logging com integração Sentry |
| `env.js` | Configura env vars a partir de ficheiros `.env.*` |
| `importer.js` | Loadable extractors para code-splitting (node e web builds) |
| `wellKnownRouter.js` | Rotas `/.well-known/*` (OAuth challenge, autenticação) |
| `metricsServer.js` | Servidor de métricas (Prometheus/observabilidade) |

### `server/api/` - 23 endpoints

Ver [Endpoints](endpoints) para a lista detalhada. Resumo:

- **Sharetribe Auth bridges**: login-as, transaction-line-items, initiate-privileged, transition-privileged, delete-account
- **Forms**: contact, newsletter, report-listing, report-user, feedback, waitlist
- **PWA**: pwa-counter (get e increment)
- **Sistemas custom V1H**: listing-like, dismiss-favorite-alerts, user-follow, dismiss-follow-alerts, dismiss-extra-alert, translate, recompute-listing-rating, host-stats
- **Auth providers**: auth/facebook, auth/google (callback)

### `server/jobs/` - 3 cron jobs

| Job | Schedule | Função |
|---|---|---|
| `notifyFavoritesJob` | `0 8 * * *` (diário 08:00) | Email resumo diário de favoritos |
| `notifySavedSearchesJob` | `*/5 * * * *` (cada 5min) | Email "novos resultados na pesquisa guardada" |
| `notifyExtraAlertsJob` | `*/5 * * * *` (cada 5min) | Deteta edits e reviews - 4 alertas e email de review |

### `server/api-util/` - 18 helpers

Helpers internos do servidor que encapsulam lógica reutilizada pelos endpoints e cron jobs:
- Cache com TTL
- Currency (Decimal.js)
- Dates (moment)
- SDK proxy (Marketplace e Integration)
- Negotiation helpers
- Line items computation
- ID tokens
- Types (constants)

### `server/resources/` - rotas de recursos

| Ficheiro | Função |
|---|---|
| `sitemap.js` | Gera 3 sitemaps XML (index, listings, pages) |
| `robotsTxt.js` | Gera `robots.txt` |
| `webmanifest.js` | Gera `manifest.json` PWA |

### `server/data/`

- `pwa-counter.json` - contador de instalações PWA (bug git: ver [Problemas Conhecidos](problemas-conhecidos))

## Config (`src/config/`)

Os ficheiros de configuração centralizam todos os parâmetros do marketplace, desde a identidade visual até às regras de pesquisa e pagamento.

| Ficheiro | Propósito |
|---|---|
| `configDefault.js` | Raiz de config - agrega todos os outros e define marketplaceName, URL e currency |
| `configBranding.js` | Logo (24/36/48px), brand image, FB sharing (1200x630), Twitter (600x314), cor `#BAA38A` |
| `configLayout.js` | Variant SearchPage (map/grid), variant ListingPage (carousel/coverPhoto), aspect ratio fotos |
| `configListing.js` | **Custom fields** (enum/multi-enum/text/long/boolean), filter config, show/save labels |
| `configMaps.js` | Mapbox/Google Maps keys, provider, location defaults, geolocation |
| `configSearch.js` | Main search (keywords/location), filters, search modes |
| `configStripe.js` | Stripe publishable key, MCC (5734), max 730 dias bookings, países suportados |
| `configUser.js` | User custom fields (línguas, etc.), public/protected/private scope |
| `configAnalytics.js` | GA4, etc. |
| `configHelpers.js` | Helpers para processar config |
| `settings.js` | Definições gerais |
| `settingsCurrency.js` | Mapeamento de moedas (EUR, USD, etc.) |
| `landingPageConfigExamples.js` | Exemplos de config landing |
| `configDefaultLocationSearches.js` | Localizações padrão no mapa |

## Routing (`src/routing/`)

| Ficheiro | Conteúdo |
|---|---|
| `routeConfiguration.js` | **57 rotas** configuradas. Cada uma tem: `name`, `path`, `component` (lazy-loaded), `loadData` (SSR), `auth` (true/false), `authPage` (redirect) |
| `Routes.js` | Renderizador (React Router) |
| `redirectsConfig.js` | Mapa de redirects 301 |

### Rotas principais

| Path | Component |
|---|---|
| `/` | LandingPage |
| `/s` | SearchPage |
| `/l/:slug/:id` | ListingPage |
| `/u/:id` | ProfilePage |
| `/login`, `/signup` | AuthenticationPage |
| `/contact`, `/privacy-policy`, `/terms-of-service`, `/cookie-policy`, `/sobre-nos`, `/como-funciona` | Páginas estáticas |
| `/inbox` | InboxPage |
| `/sale/:id`, `/order/:id` | TransactionPage |
| `/manage-listings`, `/your-listings` | ManageListingsPage |
| `/manage-account`, `/account/*` | ManageAccountPage e sub-pages |
| `/payment-methods` | PaymentMethodsPage |
| `/favoritos`, `/pesquisas-guardadas`, `/a-seguir`, `/vistos-recentemente` | Páginas V1H |
| `/checkout`, `/make-offer`, `/request-quote` | Flows de transação |
| `/l/:slug/:id/edit/...` | EditListingPage (com sub-rotas para cada tab) |
| `/p/:pageId` | CMSPage (páginas hosted Sharetribe) |
