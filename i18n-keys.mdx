A V1H tem dois ficheiros de tradução principais. O ficheiro PT é o de referência para o desenvolvimento; o EN tem mais variantes porque inclui strings herdadas do template Sharetribe que não são usadas ativamente na V1H.

- `src/translations/pt.json` — **1.383 chaves** (português europeu)
- `src/translations/en.json` — **1.764 chaves** (inglês)

A diferença de tamanho é porque o EN tem mais variantes do template original Sharetribe.

## 113 secções totais

As chaves estão organizadas por **namespace** (prefixo `ComponenteName.`). Há **113 namespaces** distintos no `pt.json`. Cada namespace corresponde a um componente ou página da aplicação.

## Como adicionar nova chave

Para adicionar uma nova string traduzível, é necessário adicioná-la em ambos os ficheiros de tradução e usá-la no código via `FormattedMessage` ou o hook `useIntl`.

1. **Adicionar ao `pt.json`**:
   ```json
   "MyComponent.myKey": "Texto em português"
   ```

2. **Adicionar ao `en.json`**:
   ```json
   "MyComponent.myKey": "Text in English"
   ```

3. **Usar no código**:
   ```jsx
   <FormattedMessage id="MyComponent.myKey" />
   ```
   
   Ou com hook:
   ```jsx
   const intl = useIntl();
   const label = intl.formatMessage({ id: 'MyComponent.myKey' });
   ```

4. **Com valores dinâmicos** (placeholders):
   ```json
   "MyComponent.greeting": "Olá {name}, tens {count} mensagens"
   ```
   ```jsx
   <FormattedMessage 
     id="MyComponent.greeting" 
     values={{ name: 'João', count: 5 }} 
   />
   ```

## Sincronização PT e EN

Não há script automático que garanta paridade entre PT e EN. Para verificar o estado de sincronização, podem ser usados os seguintes comandos.

```bash
# Conta chaves
jq 'keys | length' src/translations/pt.json
jq 'keys | length' src/translations/en.json

# Chaves em EN mas não em PT
diff <(jq -r 'keys[]' src/translations/en.json | sort) <(jq -r 'keys[]' src/translations/pt.json | sort)
```

**Gap atual**: ~381 chaves em EN não estão em PT. São do template Sharetribe original (componentes não usados pela V1H). Sem impacto prático.

## Manutenção das traduções

### Sharetribe oferece serviço de tradução automática?
**Não**. O Sharetribe `Pages` (PageBuilder) tem conteúdo em CMS que pode ser editado por idioma, mas as **strings de UI** (`pt.json`, `en.json`) são responsabilidade do dev.

### `yarn run translate`
Script que ajuda a comparar PT vs EN e detetar chaves em falta. Não traduz automaticamente — apenas reporta.

### MyMemory (tradução automática)
Para **conteúdo do utilizador** (descrições, reviews), usamos MyMemory via `/api/translate`. Ver [Tradução](traducao).

## Fr/Es/De

Os ficheiros `fr.json`, `es.json`, `de.json` existem em `src/translations/` mas são **herdados do template Sharetribe** — nunca foram atualizados pela V1H. Não são oferecidos na UI (o language switcher só mostra PT/EN).

Para suportar mais idiomas, seria preciso:
1. Atualizar esses JSONs com strings V1H
2. Adicionar opções ao language switcher
3. Configurar `localeContext` para os aceitar
