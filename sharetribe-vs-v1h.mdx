A V1H **não foi construída do zero**. Partiu do [Sharetribe Web Template](https://github.com/sharetribe/web-template) (open-source, MIT) e foi adaptada/expandida no VS Code.

Esta página explica o que veio pronto vs o que foi feito.

## O que é o Sharetribe

**Sharetribe** é uma plataforma SaaS para construir marketplaces (Airbnb-likes, eBay-likes, etc.). Fornece:
- **Backend gerido** (BD, autenticação, transações, mensagens, pagamentos)
- **API e SDKs** (Marketplace SDK + Integration SDK)
- **Console** (painel admin no `console.sharetribe.com`)
- **Web Template open-source** (React + Redux + Express SSR) como ponto de partida do frontend

O Sharetribe **trata da complexidade** de marketplace (Stripe Connect, KYC, anti-fraude, payouts) — a V1H apenas customiza UI/UX e adiciona features próprias.

## ✅ O que veio do Sharetribe (não programado pela V1H)

### Backend (gerido pelo Sharetribe SaaS)
- Base de dados (utilizadores, listings, transactions, reviews, messages)
- Autenticação (email/password, social logins, JWT tokens)
- Stripe Connect (onboarding, KYC, payouts)
- Sistema de mensagens entre cliente e anfitrião
- Eventos e webhooks
- Notificações nativas (emails de transações)

### Frontend (template open-source)
- **PageBuilder + SectionBuilder** — motor universal de páginas
- **Hooks Redux** (auth, marketplaceData, user, stripe, etc.)
- **AuthenticationPage** com forms de login/signup
- **EditListingWizard** base (Details, Pricing, Availability, Photos)
- **ListingPage** base
- **OrderPanel** (date pickers, time slots, line items)
- **InboxPage** (Sales / Orders tabs)
- **TransactionPage** (ActivityFeed, ReviewForm, SendMessageForm)
- **ProfilePage** + **ProfileSettingsPage** base
- **SearchPage** com filtros + mapa Mapbox
- **Stripe Payout Page** (onboarding hosted)
- Sharetribe SDK + Integration SDK clients
- I18n (react-intl)
- CSS Modules

### Processos de transação
- `default-booking`, `default-purchase`, `default-inquiry`, `default-negotiation` — definidos pelo Sharetribe

## Arquitetura: o que vive onde

```
Sharetribe (cloud SaaS)
  ↓ API
Express server (server/) ← cron jobs + endpoints custom V1H
  ↓ SSR
React frontend (src/) ← UI customizada V1H sobre template Sharetribe
  ↓ Stripe Connect
Stripe (cloud) ← pagamentos
```

A V1H **não tem base de dados própria** — toda a persistência é via Sharetribe (incluindo metadata customizada como favoritos, seguidores, alertas pendentes).

## Vantagens desta arquitetura

- **Time-to-market rápido**: não há de construir backend nem sistema de pagamentos
- **Manutenção focada**: V1H mexe só no UX e features de marketplace
- **Conformidade**: Stripe trata de KYC, PSD2, etc.
- **Escalabilidade**: Sharetribe gere infrastructure

## Limitações

- **Dependência de fornecedor**: se o Sharetribe sobe preços ou descontinua, é difícil mudar
- **Customização limitada**: certas coisas só dão para fazer com upgrade Sharetribe Plus
- **i18n de emails** parcialmente bloqueado (resolução requer Plus)
- **Rate limits** podem ser apertados em planos baixos (ver [Problemas Conhecidos](problemas-conhecidos))
