Os perfis são onde cada utilizador se apresenta ao resto da comunidade. A V1H tem dois tipos de página de perfil — o **perfil público** (o que os outros utilizadores veem) e as **configurações do próprio perfil** (onde se editam os dados). Inclui também o **sistema de seguidores** que permite acompanhar anfitriões favoritos e ser notificado quando publicam novos espaços.

## Perfil público (`ProfilePage`)

Aparece quando outro utilizador clica no nome ou avatar de alguém. A sidebar lateral apresenta os dados sociais e profissionais do utilizador, enquanto o conteúdo principal lista os anúncios e as reviews.

### AsideContent (sidebar esquerda)

- **AvatarLarge** (foto de perfil grande, redonda)
- **Botão Seguir / A seguir** (toggle, ver [Follow](#sistema-de-seguidores))
- **Rating e review count** - só conta reviews recebidas como **cliente** (`type: ofCustomer`). Decisão das responsáveis em 2026-05 - ver [Decisões Importantes](decisoes-importantes)
- **Followers count** (com override server-side)
- **Línguas faladas** (com bandeiras flag-icons, via `LanguagesField`)
- **Social links** (Facebook, Instagram, etc.) via `SocialLinksField`
- **Response time** formatado (ex: "Responde em ~2h")
- **Last online** relativo:
  - "Online agora"
  - "há X minutos"
  - "há X horas"
  - "há X dias"
- **Localização** (endereço)

### Main content
- Lista dos anúncios deste utilizador (com `ProfileListingsSearchBar` para filtrar)
- Reviews recebidas
- Bio

## Edição de perfil próprio (`ProfileSettingsPage`)

Ficheiro: `src/containers/ProfileSettingsPage/`

### Form: `ProfileSettingsForm`

Todos os campos têm validação client-side antes de enviar para o servidor. As regras de validação garantem consistência dos dados de perfil público.

| Campo | Validação |
|---|---|
| `firstName` / `lastName` | Letras, acentos, apóstrofos e hífens, máx 50 chars |
| `displayName` | Alfanumérico e símbolos, máx 60 chars |
| `bio` | Textarea, máx 500 chars |
| `profileLocation` | Autocomplete Mapbox |
| `avatar` | Upload com transient state |
| **Línguas faladas** | Multi-select via `LanguagesField` |
| **Social links** | Array via `SocialLinksField` (URL normalization) |
| Custom user fields | Configurados em `userFields` |

### LanguagesField

Componente custom com lista de idiomas e bandeiras (biblioteca `flag-icons`). Os idiomas disponíveis são:
- Português
- English
- Español
- Français
- Italiano
- Deutsch
- e outros

### SocialLinksField

Lista editável de redes sociais com URL normalization automática (ex: `@user` converte para `https://instagram.com/user`). Plataformas suportadas:
- Facebook
- Instagram
- Twitter
- LinkedIn
- TikTok
- YouTube
- Pinterest

## AccountSettingsPage

Página separada (`/account-settings`) para configurações de segurança e conta. A eliminação de conta é uma operação irreversível gerida por endpoint dedicado no servidor.

- Mudar email
- Mudar password
- Configurações de privacidade
- Eliminar conta (via `/api/delete-account`)

## Sistema de Seguidores

O sistema de seguidores permite acompanhar anfitriões e receber notificações quando publicam anúncios novos. O throttle de 24h garante que o anfitrião não recebe spam de notificações quando é seguido repetidamente.

Ficheiros:
- `src/ducks/follow.duck.js` - Redux state e thunks
- `server/api/user-follow.js` - endpoint para registar follow, throttle 24h e email
- `src/components/FollowAlert/FollowAlert.js` - toast in-app

### Onde se vê
- **Lista pública de seguidores** no perfil
- **Página `/a-seguir` (FollowingPage)** - lista de quem o utilizador segue
- **Sidebar do ProfilePage** - botão Seguir / A seguir
- **UserCard na ListingPage** - botão Seguir

### Funcionamento
1. Carrega "Seguir"
2. `toggleFollowAndSync(targetUserId)` no Redux
3. POST `/api/user-follow` - servidor adiciona entry em `metadata` do target, envia email e cria toast
4. Throttle: mesmo fã só notifica 1x por 24h

## Custom user fields

Configurados via `userFields` no `src/config/configUser.js`. O campo `pub_Location` é excluído da edição para evitar duplicação com `profileLocation`.

- `pub_Location` - excluído da edição para evitar duplicação com `profileLocation`

Cada campo pode aparecer no perfil público e/ou na edição.
