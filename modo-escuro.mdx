A V1H tem **modo claro** (default) e **modo escuro** alternável pelo topbar. A implementação usa dois atributos no elemento `<body>` para cobrir componentes com variáveis CSS e componentes com cores hardcoded.

## Ficheiro principal

`src/styles/landingDark.css` (~200+ linhas) - overrides globais para modo escuro.

## Toggle

Componente `DarkModeToggle` no `TopbarDesktop`. Ao clicar:
1. Alterna estado em `useDarkMode()` context
2. Adiciona/remove **2 atributos** no `<body>`:
   - `data-landing-dark="true"` - overrides de **CSS variables**
   - `data-dark-mode="true"` - overrides **hardcoded patches**
3. Persiste preferência via `localStorage` (se consent "preferences" dado)

## Estratégia dual-attribute

A necessidade de dois atributos distintos surge porque a codebase tem componentes que usam variáveis CSS e componentes que têm cores hardcoded. Cada atributo serve uma estratégia de override diferente.

### `data-landing-dark` (variáveis CSS)
Para componentes que usam **variáveis** (`var(--colorWhite)`, etc.):
- Em modo escuro, `--colorWhite` passa a ser `#1e1e1e`
- Todos os componentes que herdam reagem automaticamente

### `data-dark-mode` (patches hardcoded)
Para componentes que têm **cores hardcoded** (`background-color: #ffffff` literal):
- Selector `body[data-dark-mode] [class*="X"] { ... !important }`
- Override manual por componente

## Logo em modo escuro

Logo PNG preto é invertido via CSS filter:
```css
body[data-dark-mode] [class*="logoImage"] {
  filter: invert(1) !important;
}
```

Resultado: logo branco em modo escuro automaticamente.

## CSS Modules vs landingDark.css

A maioria dos componentes usa **CSS Modules** (`*.module.css`) com classes hashed (`MyComponent_root__abc123`). O `landingDark.css` usa seletores de substring para referenciar essas classes dinamicamente.

Para o `landingDark.css` (global) referenciar essas classes, usa o seletor `[class*="MyComponent_root"]` (matches qualquer hash).

Cuidado: `[class*="X"]` matches **substring** - pode apanhar classes não desejadas. Para precisão, usa `[class*="X__"]` (com duplo underscore antes do hash) ou `[class*="X_root"]`.

## Adicionar suporte dark a um novo componente

Para adicionar suporte a modo escuro num componente novo, segue estes passos. O teste em ambos os modos é essencial para garantir que não há elementos invisíveis ou ilegíveis.

1. Identifica as cores hardcoded
2. Adiciona override em `landingDark.css`:
   ```css
   body[data-dark-mode] [class*="MyComponent_root"] {
     background: #1e1e1e !important;
     color: #f0ece6 !important;
   }
   ```
3. Testa o componente em ambos os modos
4. Para hovers, adiciona `:hover` no selector
