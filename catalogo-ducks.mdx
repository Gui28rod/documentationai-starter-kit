## Padrão dos ducks de notificação

`favoriteAlerts`, `followAlerts`, `extraAlerts` seguem o mesmo padrão:

1. Source de verdade: `user.metadata.unseen*Alerts` (server)
2. Poll a cada 60s (`fetchCurrentUser` + extract alerts)
3. State session-level com `dismissedIds` (anti-flicker)
4. Cap visual `MAX_VISIBLE = 6`
5. Dismiss per-entry sincroniza via `/api/dismiss-*-alerts`

## Padrão dos ducks de favoritos/follow

`favorites`, `follow`:

1. Source: `user.privateData.favorites` / `user.privateData.following`
2. localStorage para guests
3. Migração automática ao login
4. Optimistic update no Redux + sync server em background
5. POST endpoints registam para alertas/throttle

## Padrão de organização

```
ducks/X.duck.js          → Redux slice + thunks
util/X.js                → helpers stateless
components/X/X.js        → componente React
components/X/X.module.css → estilo scoped
containers/X/X.js        → página completa
containers/X/X.duck.js   → estado específico da página (se necessário)
```
